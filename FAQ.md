# ❓ FAQ - Часто задаваемые вопросы

## 🚀 Запуск и установка

### В: Какие версии Java поддерживаются?
**О:** Требуется Java 17 или выше. Java 21 также поддерживается.

### В: Нужно ли устанавливать Maven отдельно?
**О:** Нет! Проект включает Maven Wrapper (`mvnw.cmd`). Просто используйте:
```bash
mvnw.cmd spring-boot:run
```

### В: Где скачать PostgreSQL?
**О:** Официальный сайт: https://www.postgresql.org/download/
Рекомендуем версию 15 или новее.

### В: Приложение не запускается, что делать?
**О:** Проверьте:
1. Запущен ли PostgreSQL (порт 5432)
2. Правильный ли пароль в `application.properties`
3. Создана ли база данных `learning_progress_tracker`
4. Свободен ли порт 8080

## 🔐 Безопасность и JWT

### В: Как долго действует JWT токен?
**О:** По умолчанию 24 часа (86400000 мс). Можно изменить в `application.properties`:
```properties
jwt.expiration=86400000
```

### В: Нужно ли менять секретный ключ JWT?
**О:** Да! Обязательно измените для продакшена:
```properties
jwt.secret=ВАШ_УНИКАЛЬНЫЙ_СЕКРЕТНЫЙ_КЛЮЧ_МИНИМУМ_32_СИМВОЛА
```

### В: Как добавить роль пользователю?
**О:** При регистрации указывайте роль в поле `role`:
- `STUDENT` - студент
- `INSTRUCTOR` - преподаватель  
- `ADMIN` - администратор

### В: Можно ли изменить роль после регистрации?
**О:** Да, но только через базу данных или дополнительный API эндпоинт (нужно добавить).

## 🌐 API и эндпоинты

### В: Как получить JWT токен?
**О:** 
1. Зарегистрируйтесь: `POST /api/auth/register`
2. Войдите: `POST /api/auth/login`
3. В ответе получите токен в поле `token`

### В: Как использовать JWT токен в запросах?
**О:** Добавьте заголовок:
```
Authorization: Bearer ВАШ_ТОКЕН_ЗДЕСЬ
```

### В: Какие эндпоинты доступны без токена?
**О:** Только публичные:
- `POST /api/auth/register`
- `POST /api/auth/login`
- `GET /` (главная страница)
- Статические файлы (CSS, JS, images)

### В: Как тестировать API?
**О:** Используйте:
- **Curl** (командная строка)
- **Postman** (GUI приложение)
- **VS Code REST Client** (расширение)
- **Swagger** (можно добавить)

### В: Где примеры API запросов?
**О:** В файле `API_EXAMPLES.md` - там 40+ готовых примеров!

## 🗄️ База данных

### В: Какие таблицы создаются автоматически?
**О:** 
- `users` - пользователи
- `courses` - курсы
- `enrollments` - записи на курсы
- `assignments` - задания
- `grades` - оценки
- `attendance` - посещаемость

### В: Нужно ли создавать таблицы вручную?
**О:** Нет! Hibernate создаст их автоматически при первом запуске.

### В: Как просмотреть данные в базе?
**О:** Используйте:
- **pgAdmin 4** (устанавливается с PostgreSQL)
- **DBeaver** (универсальный клиент)
- **Docker pgAdmin** (если используете Docker)

### В: Как сбросить базу данных?
**О:** 
**Вариант 1:** Измените в `application.properties`:
```properties
spring.jpa.hibernate.ddl-auto=create
```
После запуска верните обратно на `update`!

**Вариант 2:** Через pgAdmin:
1. Удалите базу `learning_progress_tracker`
2. Создайте заново
3. Перезапустите приложение

### В: Где хранятся пароли?
**О:** Пароли хешируются через BCrypt и хранятся в поле `password` таблицы `users`. Исходный пароль восстановить невозможно!

### В: Как переключить язык интерфейса?
**О:** В текущей версии язык по умолчанию - русский. Для изменения:

spring.web.locale=en  # английский

**О:** В файлах:
- `messages_en.properties` - английский

**О:** Можно добавить! Создайте файл `messages_fr.properties` для французского, например.

## 🎓 Использование системы

### В: Как записать студента на курс?
**О:** 
```bash
POST /api/enrollments?studentId=1&courseId=1
```

### В: Может ли студент записаться на несколько курсов?
**О:** Да! Нет ограничений на количество курсов.

### В: Как выставить оценку?
**О:** Только преподаватель или админ:
```bash
POST /api/grades
{
  "enrollmentId": 1,
  "assignmentId": 1,
  "score": 95
}
```

### В: Что означают статусы записей?
**О:**
- `ACTIVE` - студент активно учится
- `COMPLETED` - курс завершен
- `DROPPED` - студент отписался

### В: Какие статусы посещаемости?
**О:**
- `PRESENT` - присутствовал
- `ABSENT` - отсутствовал
- `LATE` - опоздал
- `EXCUSED` - отсутствовал по уважительной причине

## 🔧 Разработка и кастомизация

### В: Как добавить новый эндпоинт?
**О:**
1. Добавьте метод в соответствующий Controller
2. Добавьте бизнес-логику в Service
3. При необходимости создайте DTO

### В: Как добавить новое поле в Entity?
**О:**
1. Добавьте поле в Entity класс
2. Перезапустите приложение
3. Hibernate обновит таблицу автоматически

### В: Где изменить порт приложения?
**О:** В `application.properties`:
```properties
server.port=8081
```

### В: Как включить SQL логи?
**О:** Уже включены! Смотрите в консоли. Для отключения:
```properties
spring.jpa.show-sql=false
```

### В: Как добавить Swagger документацию?
**О:** Добавьте в `pom.xml`:
```xml
<dependency>
    <groupId>org.springdoc</groupId>
    <artifactId>springdoc-openapi-starter-webmvc-ui</artifactId>
    <version>2.2.0</version>
</dependency>
```
После запуска: http://localhost:8080/swagger-ui.html

## 🐳 Docker

### В: Как запустить только базу данных?
**О:**
```bash
docker-compose up -d postgres
```

### В: Как запустить всё в Docker?
**О:**
1. Соберите JAR: `mvnw.cmd clean package`
2. Раскомментируйте секцию `app` в `docker-compose.yml`
3. Запустите: `docker-compose up -d`

### В: Как посмотреть логи Docker?
**О:**
```bash
docker-compose logs -f app
```

### В: Как остановить контейнеры?
**О:**
```bash
docker-compose down
```

## 💾 Backup и восстановление

### В: Как сделать backup базы данных?
**О:** Через pgAdmin:
1. Правый клик на БД → Backup
2. Выберите формат и путь
3. Нажмите Backup

### В: Как восстановить данные?
**О:** 
1. Правый клик на БД → Restore
2. Выберите файл backup
3. Нажмите Restore
